# 需求文档

## 简介

Admin Dashboard 是 ShipFree SaaS 模板的基础管理后台功能模块，为应用管理员提供用户管理和数据概览能力。本模块基于现有的 Next.js App Router 架构，采用 Shadcn UI 组件库（Radix UI + TailwindCSS），遵循项目服务端优先原则，通过独立的管理员路由组实现后台管理功能。

管理后台包含两个核心功能：数据概览仪表板（统计用户数、订阅数、付款数等关键指标）和用户列表管理页（分页展示用户信息、支持搜索过滤）。管理员身份通过角色字段进行标识和权限控制，非管理员用户尝试访问时重定向至普通 dashboard。

## 需求

### 需求 1：管理员身份认证与授权

**目标：** 作为系统管理员，我希望管理后台有独立的权限控制，以便只有被授权的管理员账户才能访问管理功能，保障系统安全。

#### 验收标准

1. 当未登录用户访问 `/admin` 路径时，系统应重定向至 `/login` 页面
2. 当已登录但非管理员用户访问 `/admin` 路径时，系统应重定向至 `/dashboard` 页面并显示无权限提示
3. 当管理员用户成功登录后访问 `/admin` 时，系统应正常展示管理后台界面
4. 系统应通过用户表的 `role` 字段（值为 `admin`）识别管理员身份
5. 管理后台 layout 应在服务端进行权限验证，不可绕过客户端检查

### 需求 2：数据概览仪表板

**目标：** 作为管理员，我希望在进入管理后台后能看到关键业务指标的汇总数据，以便快速了解平台运营状况。

#### 验收标准

1. 当管理员访问 `/admin` 页面时，系统应展示以下统计卡片：总用户数、本月新增用户数、活跃订阅数、总收入
2. 当数据加载中时，系统应显示 Skeleton 占位组件，避免布局抖动
3. 当统计数据加载完成时，系统应以数字卡片形式（Shadcn Card 组件）清晰展示各项指标，并附带对应图标
4. 系统应在服务端查询数据库获取统计数据，不通过客户端 API 请求
5. 系统应显示最近 30 天的新增用户趋势（简单数字对比，无需图表）

### 需求 3：用户列表管理

**目标：** 作为管理员，我希望能查看所有注册用户的列表，并支持搜索和分页，以便高效管理用户。

#### 验收标准

1. 当管理员访问 `/admin/users` 页面时，系统应以表格形式展示用户列表，包含：用户头像、姓名、邮箱、注册时间、邮箱验证状态
2. 当用户列表超过 20 条时，系统应显示分页控件，支持跳转至指定页
3. 当管理员在搜索框输入关键词时，系统应按姓名或邮箱进行过滤，并通过 URL 查询参数保存搜索状态
4. 系统应在服务端处理分页和搜索逻辑，查询参数通过 `page` 和 `q` 传递
5. 当用户列表为空或搜索无结果时，系统应显示对应的空状态提示组件

### 需求 4：管理后台导航布局

**目标：** 作为管理员，我希望管理后台有清晰的导航结构，以便在不同管理功能间快速切换。

#### 验收标准

1. 管理后台应有独立的 Layout，包含侧边栏导航（桌面端）和顶部导航栏
2. 侧边栏应包含以下导航项：概览（`/admin`）、用户管理（`/admin/users`），并高亮当前激活路由
3. 导航栏应显示当前登录管理员的姓名/邮箱和退出登录按钮
4. 在移动端，侧边栏应折叠为汉堡菜单，点击后展开抽屉式导航
5. 系统应使用 Shadcn UI 的 `Sheet` 组件实现移动端抽屉导航

### 需求 5：Shadcn UI 组件集成

**目标：** 作为开发者，我希望管理后台统一使用 Shadcn UI 组件，以便与产品规划的 UI 框架迁移方向保持一致，并获得良好的组件质量和可访问性。

#### 验收标准

1. 管理后台所有 UI 组件（卡片、表格、按钮、分页、输入框）均应使用 Shadcn UI 组件或基于其封装
2. 项目中已存在的 `src/components/ui/` 目录下的组件（card、table、button、input、skeleton、badge、sheet 等）应被直接复用
3. 如需使用尚未安装的 Shadcn 组件，任务文档中应标注需要执行的安装命令
4. 组件样式应遵循项目现有 TailwindCSS 4 配置，使用 `cn()` 处理条件类名
